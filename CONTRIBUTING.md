# Contributing to Claude LabBook

Thanks for your interest in contributing to Claude LabBook!

## Getting Started

1. Fork the repo and clone it locally
2. Install dependencies:
   ```bash
   npm install
   ```
3. Build the project:
   ```bash
   npm run build
   ```

## Development

### Building

```bash
npm run build          # compile TypeScript
npm run dev            # watch mode (recompile on changes)
```

### Project Structure

- `src/` — TypeScript source code
- `src/db/` — database layer (Kuzu graph + LanceDB vector)
- `src/scanner/` — codebase indexing (walker, chunker, language detection)
- `src/tools/` — MCP tool implementations
- `dist/` — compiled JavaScript (generated by `npm run build`)
- `docs/` — documentation

### Code Style

- TypeScript strict mode is enabled
- ESM modules (`"type": "module"`)
- Target: ES2022, Node16 module resolution

## Submitting Changes

1. Create a feature branch from `main`
2. Make your changes with clear, focused commits
3. Ensure the project builds cleanly: `npm run build`
4. Open a pull request with a description of what and why

## Reporting Issues

Open an issue at [github.com/anthonylee991/claude-labbook/issues](https://github.com/anthonylee991/claude-labbook/issues) with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Node.js version and OS

## Areas for Contribution

- New MCP tools for experiment tracking
- Improved code chunking strategies
- Additional language support in the scanner
- Documentation improvements
- Bug fixes
- Performance optimizations
